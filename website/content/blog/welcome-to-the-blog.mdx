This is the first post on kev.dev's blog — mostly a test of the plumbing,
but also a quick note on how this section works, in case future-me forgets.

## No database, no accounts

Every post here is just a Markdown/MDX file sitting in `content/blog/`. There's
no CMS, no login, no API route hitting a database. To publish something new,
I run:

```
npm run new-post -- "My New Post Title"
```

which drops a pre-filled `.mdx` file into `content/blog/` with today's date
and a `draft: true` flag. I write the post, flip `draft` to `false`, commit,
and push — the static build picks it up automatically.

Posts are sorted by date, newest first. If I ever publish more than one on
the same day, there's an optional `order` field in the frontmatter —
higher numbers show up first among posts sharing a date, so I'm not stuck
relying on whatever order the filesystem happens to hand back.

## Why this approach

A few reasons this fits better than a full CMS:

- **Zero moving parts.** No auth, no database, nothing that can go down.
- **Git is the editor history.** Every edit is a commit; rollback is `git revert`.
- **It's just files.** Easy to back up, easy to search, easy to migrate later
  if I ever want to.
